AMD Rapido

Server Hardware Information Collection & Comparison Tool

Version 2.2

Overview

AMD Rapido is a comprehensive server hardware information collection and comparison toolkit designed specifically for AMD GPU-equipped systems. It provides detailed hardware profiling including CPU, GPU, networking, BMC, ROCm runtime, and GPU microbenchmark capabilities with an intuitive HTML-based comparison interface.

Key Components:
  • rapido-collect.py - Collects hardware information and exports to JSON
  • rapido-report.py - Generates HTML comparison reports from JSON files
  • gpu_p2p_bandwidth.cpp - GPU-to-GPU peer-to-peer bandwidth benchmark (HIP)
  • gpu_kernel_benchmarks.cpp - Comprehensive GPU compute kernel benchmarks (GEMM, convolution, memory bandwidth)

Requirements

  • Python 3.6 or higher
  • Linux operating system (primary support for AMD GPU features)
  • ROCm installation (for AMD GPU details and microbenchmarks)
  • amd-smi tool (for comprehensive GPU information)
  • ipmitool (optional, for BMC information)
  • ethtool (optional, for enhanced network details on Linux)
  • hipcc compiler (optional, for GPU P2P bandwidth testing)
  • Administrative privileges may be required for some hardware queries
Note: While the tool supports Windows and macOS for basic CPU/GPU information, advanced AMD GPU features, ROCm details, BMC information, and GPU P2P testing are only available on Linux systems.

Installation

No installation required. Simply ensure Python 3 is installed and the scripts have execute permissions:

# Make scripts executable chmod +x rapido-collect.py rapido-report.py # Verify Python version python3 --version # Install optional tools for enhanced features # For BMC information (IPMI) sudo apt-get install ipmitool # Debian/Ubuntu sudo yum install ipmitool # RHEL/CentOS # For enhanced network details sudo apt-get install ethtool # Debian/Ubuntu sudo yum install ethtool # RHEL/CentOS # For GPU benchmarks (requires ROCm and hipcc) # Compile the P2P bandwidth benchmark hipcc -o gpu_p2p_bandwidth gpu_p2p_bandwidth.cpp # Compile the kernel benchmarks hipcc -O3 -o gpu_kernel_benchmarks gpu_kernel_benchmarks.cpp

Usage

1. Collecting Hardware Information

Run rapido-collect.py on each server you want to profile:

Basic Collection

# Collect all hardware information (default) python3 rapido-collect.py # Explicit all sections collection python3 rapido-collect.py -a # Output will be saved as: serverinfo_<hostname>.json # Example: serverinfo_gpu-server-01.json

Selective Collection NEW

# Collect specific sections only (faster) python3 rapido-collect.py -c # CPU only python3 rapido-collect.py -g # GPU only python3 rapido-collect.py -n # Network only python3 rapido-collect.py -b # BMC only (may require sudo) python3 rapido-collect.py -r # ROCm only python3 rapido-collect.py -m # Microbenchmarks only (includes ROCm) # Combine multiple sections python3 rapido-collect.py -c -g # CPU and GPU only python3 rapido-collect.py -c -g -n # CPU, GPU, and Network only python3 rapido-collect.py -g -r # GPU and ROCm only

With GPU Microbenchmarks

# Include GPU peak performance calculations and kernel benchmarks python3 rapido-collect.py -m # Include microbenchmarks + GPU P2P bandwidth testing python3 rapido-collect.py -m -p

Verbose Mode NEW

# Show tool availability check and detailed progress python3 rapido-collect.py -v # Combine with other flags python3 rapido-collect.py -c -g -v # Verbose CPU and GPU collection sudo python3 rapido-collect.py -a -p -v # Full verbose collection

With BMC Information

# Collect BMC data (may require sudo for IPMI access) sudo python3 rapido-collect.py # Or collect only BMC data sudo python3 rapido-collect.py -b
Output: The script automatically generates a JSON file named serverinfo_<hostname>.json containing all collected hardware information.

2. Tool Availability Check NEW

When rapido-collect.py runs, it automatically checks for all required and optional tools and reports:

  • Which tools are available/missing (✓/✗ indicators)
  • Impact on each report section if tools are missing
  • Special checks for ROCm installation and benchmark source files
  • Summary of affected functionality
Example Output: The tool will display a comprehensive availability report showing CPU, GPU, Network, BMC, ROCm, and Microbenchmark tools status before starting data collection.

3. Generating Comparison Reports

Use rapido-report.py to generate HTML comparison reports:

Compare Two Servers

# Compare two servers side-by-side python3 rapido-report.py \ -f1 serverinfo_server1.json \ -f2 serverinfo_server2.json \ -o comparison_report.html

Single Server Report

# Generate report for a single server python3 rapido-report.py \ -f1 serverinfo_server1.json \ -o server1_report.html

Legacy Mode (Single File)

# Using legacy -i flag python3 rapido-report.py \ -i serverinfo_server1.json \ -o report.html

Command-Line Options

rapido-collect.py

Option Description
No arguments Collects all hardware information (CPU, GPU, Network, BMC, ROCm) - same as -a
-a, --all Collect all sections (CPU, GPU, Network, BMC, ROCm)
-c, --cpu Collect CPU information only
-g, --gpu Collect GPU information only
-n, --network Collect Network information only
-b, --bmc Collect BMC information only (may require sudo)
-r, --rocm Collect ROCm information only
-m, --microbenchmarks Collect microbenchmarks only (GPU peak performance and kernel benchmarks, automatically includes ROCm)
-p, --p2p Include GPU-to-GPU P2P bandwidth tests (requires -m flag)
-v, --verbose Show tool availability check and detailed progress messages
Note: Selective collection flags (-c, -g, -n, -b, -r, -m) can be combined. When any specific flag is used, only those sections are collected. The report will only show tabs for sections with data.
Note: The -p flag requires the -m flag to be effective. Use python3 rapido-collect.py -m -p to enable both microbenchmarks and P2P testing.

rapido-report.py

Option Description Default
-f1, --file1 Path to the first JSON file for comparison None
-f2, --file2 Path to the second JSON file for comparison None
-i, --input Path to a single JSON input file (legacy mode) None
-o, --output Path for the generated HTML report file comparison.html

Output Description

JSON Output (rapido-collect.py)

The JSON file contains structured hardware information organized into sections:

  • _metadata - Collection metadata (date, status, errors) NEW
  • cpu - CPU information (manufacturer, cores, threads, frequency, cache)
  • gpu - GPU details (name, VRAM, compute units, drivers, architecture, ASIC info, PCIe, VBIOS, power/thermal limits, cache, NUMA, RAS, firmware)
  • network - Network interfaces (name, state, MAC/IP addresses, speed, duplex, driver/firmware, PCI address, RX/TX stats)
  • bmc - BMC information (device info, network config, sensors, FRU data, system events, chassis status)
  • rocm - ROCm runtime details (version, packages, environment, modules)
  • microbenchmarks - GPU peak performance, kernel benchmarks, and P2P bandwidth measurements
Metadata Tracking: Each JSON file now includes a _metadata section with collection timestamp, status (complete/partial), and any errors encountered during collection.

HTML Report (rapido-report.py)

The HTML report features an interactive tabbed interface with:

  • CPU Tab - CPU specifications and capabilities
  • GPU Tab - Comprehensive AMD GPU information via amd-smi including:
    • ASIC details (name, architecture, device/vendor IDs, compute units, serial number)
    • Driver information
    • PCIe bus information (BDF, link speed, generation, width)
    • VBIOS information (version, part number, build date)
    • Board information (model, manufacturer, serial)
    • VRAM details (size, type, vendor, bandwidth, bit width)
    • Cache information (L1/L2/L3 sizes and instances)
    • Power and thermal limits (max/min power, temperature thresholds)
    • NUMA node and CPU affinity
    • Partition information (SR-IOV)
    • Firmware versions
    • RAS features (ECC schemas, enabled/disabled blocks)
  • Network Tab - Detailed network interface information including:
    • Interface name, state, and link detection
    • MAC address and IPv4/IPv6 addresses
    • Link speed, duplex mode, MTU
    • Driver name, version, and firmware
    • PCI address
    • RX/TX bytes and packets (with human-readable formatting)
    • Error statistics
  • BMC Tab - Baseboard Management Controller information including:
    • BMC device information (firmware version, device ID, manufacturer)
    • Network configuration (IP, MAC, subnet, gateway)
    • Temperature sensors
    • Voltage sensors
    • Fan sensors
    • Power sensors
    • FRU (Field Replaceable Unit) data
    • System Event Log (SEL) information and recent events
    • Chassis power and status information
  • ROCm Tab - ROCm runtime environment and installation details
  • Microbenchmarks Tab - GPU peak performance, kernel benchmarks, and P2P bandwidth:
    • Peak FP32, FP64, FP16, INT8 performance calculations
    • Kernel Benchmarks (NEW): GEMM FP32/FP64, memory bandwidth, vector operations, FMA throughput, 1D convolution
    • GPU-to-GPU P2P bandwidth measurements (all pairs)
    • P2P capability detection
Comparison Mode: When comparing two files, each tab displays data side-by-side for easy visual comparison between systems. The report title will show "AMD Rapido Server Information Comparison". For single-device reports, the title is "AMD Rapido Server Information".
Dynamic Tab Visibility: Tabs only appear when data is available for that section. For example, if you collected only CPU and GPU data using -c -g flags, only CPU and GPU tabs will be shown in the report.

GPU P2P Bandwidth Testing NEW

The GPU peer-to-peer bandwidth benchmark measures direct GPU-to-GPU communication speeds:

How It Works

  • Tests all GPU pairs in the system (bidirectional)
  • Uses 256 MB transfers over 10 iterations for accurate measurements
  • Checks P2P capability between each GPU pair
  • Reports bandwidth in GB/s for each connection

Compiling the Benchmark

# Compile with hipcc (requires ROCm installation) hipcc -o gpu_p2p_bandwidth gpu_p2p_bandwidth.cpp # The compiled binary is automatically used by rapido-collect.py python3 rapido-collect.py -m -p

Output Example

P2P bandwidth results appear in the Microbenchmarks tab showing:

  • Source GPU and destination GPU
  • GPU architecture names
  • P2P enabled/disabled status
  • Bandwidth in GB/s

Troubleshooting

Common Issues

Issue: No AMD GPU information collected
Solution: Ensure ROCm is installed and amd-smi is accessible. Check if amd-smi list works. Fallback to rocm-smi if available.
Issue: BMC tab not appearing in report
Solution: Install ipmitool and run collection with sudo. Verify IPMI is accessible: sudo ipmitool bmc info
Issue: GPU P2P bandwidth shows 0.00 GB/s
Solution: Compile the HIP benchmark (hipcc -o gpu_p2p_bandwidth gpu_p2p_bandwidth.cpp). Ensure P2P is supported between GPUs. Check PCIe topology.
Issue: Permission denied errors
Solution: Some hardware queries require elevated privileges. Try running with sudo python3 rapido-collect.py
Issue: Missing network driver/firmware information
Solution: Install ethtool for enhanced network details: sudo apt-get install ethtool (Debian/Ubuntu) or sudo yum install ethtool (RHEL/CentOS)
Issue: GPU data parsing errors
Solution: Ensure amd-smi is up to date. The tool supports both old and new amd-smi JSON formats. Check that amd-smi static --json works manually.
Issue: JSON file corrupted or incomplete
Solution: This usually means data collection was interrupted (Ctrl+C, crash, or system shutdown). Re-run data collection: python3 rapido-collect.py. The tool now includes crash protection and will save partial data if interrupted.
Issue: Empty tabs appearing in report
Solution: Use selective collection flags to only collect needed sections. Example: python3 rapido-collect.py -c -g for CPU and GPU only.

Best Practices

  • Run collection during system idle time for accurate baseline measurements
  • Use sudo for complete hardware access (especially for BMC and some GPU features)
  • Compile GPU P2P benchmark before running collection with -p flag
  • Keep JSON files organized with clear naming conventions (hostname, date, etc.)
  • Archive JSON files for historical comparison and tracking hardware changes
  • Use descriptive output filenames when generating comparison reports
  • Verify ROCm, GPU drivers, and amd-smi are up to date before collection
  • Install optional tools (ipmitool, ethtool) for comprehensive data
  • For multi-GPU systems, always use -m -p flags to capture P2P performance

Performance Considerations

  • Basic collection (~5-10 seconds) - CPU, GPU, Network, BMC, ROCm info
  • Selective collection (~1-5 seconds) - Only specific sections (e.g., -c -g)
  • With microbenchmarks (-m) (~20-40 seconds) - Adds peak performance calculations and kernel benchmarks
  • With P2P testing (-m -p) (~30-120 seconds) - Depends on number of GPUs (N² tests for N GPUs)
  • BMC collection (~10-30 seconds) - Depends on IPMI responsiveness
Tip: Use selective collection flags (-c, -g, -n, etc.) to significantly reduce collection time when you only need specific hardware information.
